Keep It Simple

  • Try the most simple approach that will work
  • Modularize: do one thing well
For machine learning: non-opaque machine learning algorithms, used unix utilities (biggest challenges were speed and scale), at a startup, product goes beyond the ML.
For data pipeline: simple, modular data pipeline (no sharing between teams)

Avoid relational databases

  • administrative hassle
  • complexity
  • performance
Now: data pipelines moving from data warehouse to a data pipeline where everyone has access. Touching DBs is hairy!!
key value stores: simple data moodel, very fast, horizontally scalable => can handle massive concurrent load

Just add a server

Compute speed has increased (double every 18 months until ~2012), but big data has been about parrellization. Good since Moore's law has switched to multicore processors and parallelism.

Measure Everything

A/B testing. Only build what you need, and measure it. This is huge in startups when you have limited resources.
